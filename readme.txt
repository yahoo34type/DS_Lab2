Лабораторная работа №2 "Разработка распределенного приложения, использующего технологию передачи сообщений и сокеты"
Разработанное приложение позволяет посылать данные сервису при помощи очередей сообщений и сокетов. Сервис осуществляет передачу полученных
данных в базу данных под управлением СУБД MySql.
Сервис поделен на две части, каждая из которых представляет собой службу ОС Windows 10.
Служба 1 осуществляет передачу данных при помощи очередей сообщений.
Служба 2 осуществляет передачу данных при помощи сокетов.
Все данные передаются при помощи синхронного ключа, известного как приложению, так и сервису.

Проверка приложения на соответствие требованиям:
1) Приложение позволяет выполнять прием и передачу данных из ненормализованной БД в нормализованную без модификации данных.
	Проверка осуществляется по следующему сценарию:
	1.1) Сформулируем контрольный пример для проверки. (см. рисунок 1)
	1.2) Выполним нормализацию КП к 3 НФ вручную, получим содержимое таблиц, которое мы должны получить в результате работы приложения. (см. рисунок 2)
	1.3) Запустим приложение и осуществим передачу данных сервису любым из способов.
	1.4) Если содержимое таблиц соответствует полученному на шаге 1.2 результату, то будем считать, что требование 1 выполнено. 
	Приложение позволяет выполнять прием и передачу данных из ненормализованной БД в нормализованную без модификации данных.
2) Приложение позволяет передавать информацию с помощью очередей сообщений.
	Проверка осуществляется по следующему сценарию:
	2.1) Выполним остановку сервиса.
	2.2) Очистим содержимое таблиц БД.
	2.3) При помощи приложения передадим данные в очередь сообщений.
	2.4) При помощи средств Windows 10 обнаружим, что сообщение появилось в очереди сообщений, однако изменений в БД не произошло.
	2.5) Запустим сервис, отвечающий за прием данных из очереди сообщений, и увидим, что сообщения пропали из очереди.
	2.6) Если таблицы оказались заполнены в соответствии с контрольным примером, то будем считать, что требование 2 выполнено.
	Приложение позволяет передавать информацию с помощью очередей сообщений.
3) Приложение позволяет передавать информацию с помощью сокетов.
	Проверка осуществляется по следующему сценарию:
	3.1) Выполним остановку сервиса.
	3.2) Очистим содержимое таблиц БД.
	3.3) При помощи приложения передадим данные при помощи сокетов.
	3.4) Увидим, что никаких изменений в БД не произошло.
	3.5) Запустим сервис, отвечающий за прием данных при помощи сокетов.
	3.6) При помощи приложения снова попытаемся передать данные при помощи сокетов.
	3.7) Если таблицы оказались заполнены в соответствии с контрольным примером, то будем считать, что требование 3 выполнено.
	Приложение позволяет передавать информацию с помощью сокетов.
4) При передаче данных они шифруются с помощью симметричного ключа.
	Проверка осуществляется по следующему сценарию:
	4.1) Выполним остановку сервиса.
	4.2) Передадим данные при помощи очереди сообщений.
	4.3) При помощи средств Windows 10 обнаружим, что сообщение появилось в очереди сообщений.
	4.4) Если тело сообщения зашифровано и данными нельзя воспользоваться, то будем считать, что требование 4 выполнено.
	При передаче данных они шифруются с помощью симметричного ключа.
